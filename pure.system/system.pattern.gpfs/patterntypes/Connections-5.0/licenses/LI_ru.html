Единица Мощности Процессора (PVU)

Единица мощности процессора (PVU) — это единица измерения, по которой может лицензироваться эта Программа. Число необходимых разрешений на основе PVU зависит от процессорной технологии (определяется по Поставщику Процессора, Торговой Марке, Типу и Номеру Модели в Таблице PVU, размещенной по адресу http://www.ibm. com/software/lotus/passportadvantage/pvu_licensing_for_customers.html) и числа процессоров, доступных для Программы. Для цели лицензирования на основе PVU IBM по-прежнему определяет процессор как процессорное ядро на микросхеме. Микросхема двухъядерного процессора, например, содержит два процессорных ядра.

Лицензиат может внедрить Программу с использованием либо модели лицензирования на основе Полных Мощностей, либо модели лицензирования на основе Виртуализованных Мощностей (Неполных Мощностей) в соответствии с Условиями Лицензирования на основе Неполных Мощностей по программе Passport Advantage (см. нижеуказанную Web-страницу). В случае использования лицензирования на основе Полных Мощностей Лицензиат должен приобрести разрешения на основе PVU в объеме, достаточном для охвата всех активированных процессорных ядер* в физической аппаратной среде, доступных для Программы или управляемых ею, за исключением тех серверов, с которых Программа удалена навсегда. В случае использования лицензирования на основе Виртуализованных Мощностей Лицензиат должен приобрести разрешения в объеме, достаточном для охвата всех активированных процессорных ядер, доступных для Программы или управляемых ею, в соответствии с Правилами Подсчета числа Лицензий для Виртуализованных Мощностей на Web- странице по адресу http://www.ibm. com/software/lotus/passportadvantage/Counting_Software_licenses_using_specific_virtualization_technologies.html.

* Активированное процессорное ядро — это процессорное ядро, доступное для использования на физическом или виртуальном сервере, даже в том случае, если мощность процессорных ядер может быть ограничена или ограничивается с помощью технологий виртуализации, команд операционной системы, параметров настройки BIOS или аналогичных ограничений.

"Соотношение n/m" означает, что Лицензиат в совокупности получает определенное число ("n") разрешений на Вспомогательную Программу на каждое указанное число ("m") разрешений на Основную Программу. Если не указано иное, число разрешений на Вспомогательную Программу округляется до значения, кратного n. Например, если Программа предусматривает 100 PVU для Вспомогательной Программы на каждые 500 PVU, полученные для Основной Программы, и Лицензиат приобрел 1 200 PVU для Программы, Лицензиату разрешается установить Вспомогательную Программу и использовать число процессорных ядер, доступных для Программы или управляемых Программой, для 300 PVU. Эти единицы PVU не требуется учитывать в общем числе PVU, необходимом для установки Программы Лицензиатом в том, что касается установки Вспомогательной Программы (хотя это число PVU может потребоваться учесть по другим причинам, например для определения числа процессорных ядер, доступных также для Основной Программы).

"Использование Основной Программой" означает, что Вспомогательная Программа предоставляется исключительно для использования Основной Программой.
