Hinweise und Tipps für die Classic Express Launchpad-Schablone

Die Classic Express Launchpad-Schablone bietet Ihnen einen Ausgangspunkt für die Erstellung eines Konfigurations-Launchpads, das Sie in Verbindung mit Produkten der Marke 'Express' verwenden können.

Diese Schablone verfügt über Code, den Sie zum Starten des Installationsprogramms eines Produkts, zur Anzeige der Dokumentation und zur Bereitstellung lokalisierter Inhalte verwenden können.

Hinweise zur Classic Express Launchpad-Schablone:

  • Alle verwendeten Befehle und Programme sind in 'launchpadContent/global.properties' definiert.
  • Die Musteranwendungen sind auf DISK1 der Disk Layout-Seite enthalten. Um angepasste Installationsprogramme oder -anwendungen hinzuzufügen, definieren Sie die Anwendung in 'launchpadContent\global.properties' (nach den Beispielen für die Musteranwendungen). Importieren Sie die Anwendungen zum entsprechenden Datenträger im Projekt und verwenden Sie hierzu die Steuerelemente der Datenträger-Layout-Registerkarte.
  • Alle HTML-Dateien verwenden dasselbe Style-Sheet (content.css), das sich im Verzeichnis 'launchpadContent' befindet.

Die Classic Express Launchpad-Schablone enthält die folgenden Seiten:

  • Welcome (Willkommen) - Die Begrüßungsseite bietet in der Regel einen Überblick über das Produkt und eine Erläuterung der Funktionen des Launchpads. Die Begrüßungsseite sollte kurz gefasst sein und dem Benutzer dennoch Optionen bieten, um weitere Informationen zum Produkt zu finden, indem ein Link zur Produkt-Website bereitgestellt wird, der in einem neuen Browser-Fenster angezeigt wird. Die Begrüßungsseite verfügt über Code, mit dem Sie eine externe Website in einem Browser laden können. Verwenden Sie statt der JavaScript-Funktion window.open() die Funktion viewPage() zur Anzeige neuer Fenster, da die Wahrscheinlichkeit höher ist, dass window.open() von einem Blockierungsprogramm für Popups geblockt wird.
  • Release Information (Release-Informationen) - Die Seite mit den Release-Informationen enthält für gewöhnlich Angaben, die sich speziell auf ein bestimmtes Release eines Produkts beziehen. Sie finden dort beispielsweise Hinweise zu Installationsproblemen, geänderte Voraussetzungen, wichtige Features oder Programmkorrekturen im neuen Release und ähnliche Inhalte. Die Seite mit den Release-Informationen enthält Code für die Anzeige einer PDF-Datei und die Ausführung einer Anwendung.
  • Prerequisite Information (Informationen zu den Voraussetzungen) - Die Seite mit den Voraussetzungen enthält für gewöhnlich ein Steuerelement für die Ausführung eines Prüfprogramms für die Voraussetzungen. Dieses Programm ermittelt, ob auf dem Computer die entsprechenden Systemvoraussetzungen erfüllt werden, und liefert eine Liste der fehlenden Voraussetzungen. Die Seite mit den Informationen zu den Voraussetzungen enthält Code für die Anzeige einer PDF-Datei und die Ausführung einer Anwendung.
  • Installation Scenario - Auf der Seite mit den Installationsszenarios werden für gewöhnlich die verschiedenen Methoden beschrieben, mit denen ein Benutzer ein Produkt installieren kann. Je nach Einsatzart des Produkts können sich die Benutzer für ein bestimmtes Installationsszenario entscheiden. Die Unterschiede zwischen den verschiedenen Installationsszenarios, die dem Benutzer zur Verfügung stehen, werden erläutert. Außerdem werden die Gründe aufgeführt, die in bestimmten Situationen für oder gegen die einzelnen Installationsszenarios sprechen. Nicht alle Produkte enthalten Installationsszenarios. Wenn Sie die Classic Express Launchpad-Schablone für ein solches Produkt anpassen, können Sie diesen Abschnitt aus der Navigation entfernen. Die Seite mit den Installationsszenarios enthält Code für die Ausführung eines angepassten Programms mit einem Argument.
  • Migration Information (Migrationsinformationen) - In den Migrationsinformationen wird für gewöhnlich das Verfahren zur Umstellung erläutert, das ein Benutzer anwenden muss, um ein Upgrade von einer früheren Produktversion auf eine höhere Version durchzuführen. Wenn die Migrationsinformationen für ein bestimmtes Produkt nicht relevant sind, sollte dieser Inhalt aus der Navigation entfernt werden. Die Seite mit den Migrationsinformationen enthält Code für die Anzeige einer PDF-Datei und die Ausführung einer Anwendung.
  • Install (Installation) - Die Seite für die Produktinstallation enthält für gewöhnlich ein Steuerelement für die Einleitung einer Produktinstallation. Sie können einem Konfigurations-Launchpad mehrere Produktinstallationsseiten hinzufügen, wenn die Installationen mehrerer Produkte unterstützt werden sollen. Die Installationsseite enthält Code für den Start einer Produktinstallation.
  • Exit (Beenden) - Der Link zum Beenden auf dem Navigationsmenü beendet das Launchpad.
