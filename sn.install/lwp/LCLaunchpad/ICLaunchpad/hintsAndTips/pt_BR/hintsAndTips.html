Sugestões e Dicas para o Modelo de Painel de Ativação do Classic Express

O Modelo de Painel de Ativação do Classic Express fornece um ponto de início para criar um painel de ativação de configuração que você pode usar com produtos com a marca Express.

Este modelo possui um código que pode ser utilizado para iniciar um programa de instalação do produto, exibir a documentação e fornecer o conteúdo localizado.

Coisas a serem observadas sobre o Modelo de Painel de Ativação do Classic Express:

  • Todos os comandos e programas utilizados são definidos em launchpadContent/global.properties.
  • Os aplicativos de amostra estão inclusos em DISK1, na página Layout do Disco. Para incluir programas ou aplicativos de instalação customizados, defina o aplicativo em launchpadContent\global.properties (seguindo os exemplos para os aplicativos de amostra). Importe os aplicativos no disco apropriado do projeto, utilizando os controles localizados na guia de layout do disco.
  • Todos os arquivos HTML utilizam a mesma folha de estilo (content.css) que reside no diretório launchpadContent.

O Modelo de Painel de Ativação do Classic Express contém as seguintes páginas:

  • Bem-vindo - Geralmente, a página de boas-vindas apresenta uma visão geral do produto e uma explicação dos recursos do launchpad. A página de boas-vindas deve ser concisa, porém deve conter as opções do usuário para localização de informações adicionais sobre o produto, apresentando um link para o Web site do produto que é exibido em uma nova janela do navegador. A página de boas-vindas possui código para exibir um Web site externo em um navegador. Utilize a função viewPage() para exibir novas janelas, em vez de window.open() do JavaScript, porque window.open() tem grandes chances de ser bloqueado por um bloqueador de pop-up.
  • Informações sobre o Release - Geralmente, a página de informações sobre o release apresenta informações específicas para um release exclusivo de um produto. Por exemplo, questões de instalação, alterações feitas nos pré-requisitos, principais recursos ou correções que foram tratados no release e conteúdo semelhante é apropriado para uma página de informações sobre o release. A página de informações sobre o release apresenta código para visualização de um PDF e execução de um aplicativo.
  • Informações sobre Pré-requisito - Geralmente, a página de pré-requisitos apresenta um controle para execução de um programa de verificação de pré-requisitos que determina se os pré-requisitos do sistema existem no computador, e retorna uma lista dos pré-requisitos que não são detectados. A página de informações sobre pré-requisito apresenta código para visualização de um PDF e execução de um aplicativo.
  • Cenário de Instalação - Geralmente, o conteúdo dos cenários de instalação fornece as diversas abordagens possíveis para instalação de um produto para o usuário. Os usuários podem escolher um cenário de instalação específico com base na forma como ele deseja utilizar o produto. Esse conteúdo explica as diferenças entre os diversos cenários de instalação que o usuário pode escolher e apresenta informações que explicam porque o usuário escolhe um cenário de instalação em vez de outro. Nem todos os produtos apresentam cenários de instalação. Se você estiver customizando o Modelo de Painel de Ativação do Classic Express para um produto desse tipo, é possível remover esta seção da navegação. A página de cenários de instalação apresenta código para execução de um programa customizado com um argumento.
  • Informações de Migração - Geralmente, o conteúdo de migração explica um caminho de migração que o usuário deve seguir para fazer upgrade de uma versão anterior do produto para uma versão mais recente. Se as informações de migração não forem relevantes para um produto específico, este conteúdo deverá ser removido da navegação. A página de informações de migração apresenta código para visualização de um PDF e execução de um aplicativo.
  • Instalar - Geralmente, a página de instalação do produto apresenta um controle para iniciar a instalação de um produto. Você pode incluir várias páginas de instalação do produto em um launchpad de configuração para suportar as instalações de diversos produtos. A página de instalação apresenta código para iniciar a instalação de um produto.
  • Sair - O link de saída no menu de navegação fecha o launchpad.
