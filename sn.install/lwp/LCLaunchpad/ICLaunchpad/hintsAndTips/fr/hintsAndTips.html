Astuces et conseils pour le modèle Classic Express Launchpad

Le modèle Classic Express Launchpad constitue un point de départ pour la création d'un tableau de bord d'installation que vous pouvez utiliser avec les produits de marque Express.

Ce modèle contient un code que vous pouvez utiliser pour lancer le programme d'installation d'un produit, afficher une documentation et fournir un contenu localisé.

Remarques à propos du modèle Classic Express Launchpad :

  • Tous les programmes et commandes utilisés sont définis dans le fichier launchpadContent/global.properties.
  • Les exemples d'application sont inclus sur le DISQUE1 de la page de disposition du disque. Pour ajouter des applications ou des programmes d'installation personnalisés, définissez l'application dans launchpadContent\global.properties (en suivant les exemples des exemples d'application). Importez les applications sur le disque approprié dans le projet à l'aide des commandes de l'onglet de disposition du disque.
  • Tous les fichiers HTML utilisent la même feuille de style (content.css), qui se trouve dans le répertoire launchpadContent.

Le modèle Classic Express Launchpad contient les pages suivantes :

  • Bienvenue - La page de bienvenue offre généralement une présentation du produit et une explication des fonctions du tableau de bord. Elle doit être concise, mais proposer cependant à l'utilisateur des options permettant de rechercher des informations supplémentaires sur le produit en offrant un lien vers le site Web du produit qui est affiché dans une nouvelle fenêtre de navigateur. La page de bienvenue comprend un code permettant d'afficher un site Web externe dans un navigateur. Utilisez la fonction viewPage() pour afficher les nouvelles fenêtres au lieu de la fonction JavaScript window.open() car cette dernière peut être bloquée par une fonction de blocage des fenêtres en incrustation.
  • Release Information (Informations sur la version) - La page d'informations sur la version fournit généralement des informations spécifiques à une version d'un produit. Par exemple, les difficultés liées à l'installation, les changements de prérequis, les principales fonctions ou les correctifs résolus dans la version et tout autre contenu similaire peuvent apparaître dans une page d'informations sur la version. La page d'informations sur la version contient un code permettant d'afficher un PDF et d'exécuter une application.
  • Prerequisite Information (Informations sur les prérequis) - La page des prérequis fournit généralement une commande permettant d'exécuter un programme de vérification des prérequis qui détermine si les prérequis du système se trouvent bien sur l'ordinateur et renvoie une liste de ceux qu'il n'a pas détectés. La page d'informations sur les prérequis contient un code permettant d'afficher un PDF et d'exécuter une application.
  • Installation Scenario (Scénario d'installation) - Le contenu des scénarios d'installation fournit généralement les diverses approches proposées à un utilisateur pour l'installation d'un produit. Les utilisateurs peuvent choisir un scénario d'installation spécifique en fonction de la manière dont ils souhaitent utiliser le produit. Ce contenu explique les différences entre les divers scénarios d'installation que peut choisir un utilisateur et fournit des informations sur l'intérêt pour l'utilisateur de chacun de ces scénarios d'installation. Les produits ne proposent pas tous des scénarios d'installation. Si vous personnalisez le modèle Classic Express Launchpad pour un tel produit, vous pouvez supprimer cette section de la navigation. La page des scénarios d'installation contient un code permettant d'exécuter un programme personnalisé avec un argument.
  • Migration Information (Informations sur la migration) - La page du contenu de la migration explique généralement le chemin de migration qu'un utilisateur doit suivre pour migrer d'une version antérieure d'un produit vers une version plus récente. Si les informations de migration ne s'appliquent pas à un produit particulier, ce contenu doit être supprimé de la navigation. La page d'informations sur la migration contient un code permettant d'afficher un PDF et d'exécuter une application.
  • Install (Installation) - La page d'installation du produit fournit généralement une commande permettant de lancer l'installation du produit. Vous pouvez ajouter plusieurs pages d'installation de produit à un tableau de bord d'installation pour prendre en charge les installations de plusieurs produits. La page d'installation contient un code permettant de lancer l'installation du produit.
  • Quitter - Le lien Quitter du menu de navigation permet de sortir du tableau de bord.
