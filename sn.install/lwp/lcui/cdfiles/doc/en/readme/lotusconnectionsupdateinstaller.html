IBM Connections Update Installer README

This document describes the IBM® Connections update installer application.

Last updated: Sep 22, 2011

This section describes the Connections update installer application, which can be used to install interim or cumulative fixes on the IBM Connection product. The application can also be used to uninstall fixes. This section provides examples of the command syntax used by the update installer.

This section is a reference for the updateWizard command and its parameters. For detailed instructions on using the update installer, always refer to the README file provided with the interim or cumulative fix.

Downloading the IBM Connections Update Installer

Download the UpdateInstaller.zip file which contains the IBM Connections Update Installer application from the appropriate support page:
  • <need IBM Connections customer support link>
Although the update installer files can be unpacked anywhere on the machine, we recommend placing the files in the IBMConnections_root/update directory. IBMConnections_root is your IBM Connections product installation directory, normally, this would be IBM\IBM-Connections. Interim fixes can be downloaded to the updateinstaller/fixes directory. If different directories are used, adjust the instructions accordingly.

 

Notes:

  • On Microsoft Windows platforms, the pkunzip utility might not decompress the download image correctly. Use another utility (such as WinZip) to unzip the image.

The update installer records processing results in log files in the IBMConnection_root/version/log directory. Backup files created during the installation of interim fixes are stored in the IBMConnections_root/version/backup directory. These files are required to uninstall an interim fix.

Using the updateWizard command

Do not launch multiple copies of the update installer at one time. The update installer cannot be launched concurrently with itself. Performing more than one update at the same time can lead to a failed or faulty installation.

Validate security settings for IBM Connections after performing an update. Changes you may have made to the settings could be overwritten by the update. For example, even if you explicitly disabled the All Authenticated role, the update enables it by default.

Installation roots
IBMConnections_root represents the directory where IBM Connections is installed. By default, this varies per operating system:
  • AIX: /usr/IBM/IBM-Connections
  • UNIX and Linux: /opt/IBM/IBM-Connections
  • Windows: drive:\Program Files\IBM\IBM-Connections
Space requirements
Space requirements vary depending on what you are installing. The size of each download is available on the Support site. After unpacking the ZIP file you download, delete the ZIP file to free space. Space is also required for backup files in the IBMConnections_root/version/backup directory.
Prerequisite Java environment setting
Set up the Java environment for the update installer:
  1. Open a command line window.
  2. Change directory to the app_server_root/bin directory, where app_server_root is the installation directory of the WebSphere Application Server that is associated with IBM Connections.
  3. Issue the appropriate command:
    • UNIX: .(space)./setupCmdLine.sh
      Note: When running this command in a UNIX shell, be sure to use the syntax .(space)./setupCmdLine.sh. If you do not precede the command with the period and space, the Java™ environment will not be properly set for the active shell.
    • Linux: source setupCmdLine.sh
    • Windows: setupCmdLine.bat
Command name
updateWizard.{sh|bat}

Command parameters
Given certain command parameters, the updateWizard command can:
  • Install or uninstall interim or cumulative fixes
  • Provide information about the update state of applied interim or cumulative fixes

The following table describes the parameters that can be used with the updateWizard command. Variables in the parameter column are show italicized, like this.

Parameter Description
-? Shows command usage
/?

Shows command usage on Windows platforms only. Not supported for Linux and UNIX-based platforms.
-configProperties propertyFile.properties Specifies an externally supplied properties file containing IBM Connections properties and values.
When specifying properties in a file, the following conventions should be observed:
  • Values identified for properties should not have trailing spaces
  • Do not enclose values in quotation marks
  • When typing directories, use a forward slash (/) instead of a backward slash (\), regardless of the operating system used
-fix Interim fix only. Identifies the update as an interim fix update.
-fixDetails Interim fix only. Displays interim fix detail information.
-fixDir Interim fix only. Specifies the fully qualified directory where you download interim fixes. This directory is usually the IBMConnections_root/update/fixes directory
-fixes Interim fix only. Specifies a fix to install or uninstall.
-help Shows command usage
/help Shows command usage
-install Installs the update, either interim fix or fixpack
-installDir Specifies the fully qualified installation root of the IBM Connections product
-uninstall Uninstalls the identified fix
-uninstallAll Interim fix only. Specifies to uninstall all applied interim fixes.
-usage Shows command usage
-wasUserId WebSphere Application Server administrator ID
-wasPassword WebSphere Application Server administrator password

Product version and history information
The IBMConnections_root/version directory
in the installation root contains important data about IBM Connections and
its installed components, such as the build version and build date. This information
is included in the [product].product and [component].component
files.


The IBMConnections_root/version/history directory
in the installation root contains a collection of records for installed fixes. This information is included in the [interim fixID].efixApplied, [interim
fixID].efixDriver


A driver file has useful information
about the entire contents of an interim fix. The applied file
has relevant information about the fixes that are currently applied.


The event.history
file contains a detailed log about updates you have applied, either successfully
or unsuccessfully. Time-stamped, detailed logs record each update process
in the IBMConnections_root/version/log directory
of the installation root.

Product information files


There are
two kinds of product information files: 
    
  
  • Files in the IBMConnections_root/version directory
that store version information
    • 
  
  • Files in the IBMConnections_root/version/history directory
that store version history information
    • 





Version information


The following XML files represent installed items and
installation events:

  
product-id.Product

  
This file indicates the particular IBM Connections feature
that is installed. Data in the file indicates the version, build date, and
build level. For example, the file might be named activities.product,
which indicates that the installed feature is IBM Connections Activities .
An example of the file follows:

<!DOCTYPE product SYSTEM "product.dtd">
<product name="IBM Connections activities">
  <id>activities</id>
  <version>1.0.0.0</version>
  <build-info date="08/08/2007" 
              level="08181252"/>
</product>




component-name.Component


These component files indicate the presence of an installed component,
which is part of the feature. Data in the file indicates the component build
date, build version, component name, and product version. For example, the
file might be named blogs.component, which indicates that
the IBM Connections blogs component
is installed. An example of the file follows:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE component PUBLIC "componentId" "component.dtd">
<component build-date="2007-07-15" build-version="1.0.0.0" name="blogs" spec-version="1.0"/>




fix-id.efix


These fix files indicate the presence of an installed interim fix.




History information


This file stores version
history information:
event.history


This file lists update events that have occurred. An update event is an
operation that installs or uninstalls an interim fix. The file
is sorted by the date and time of the events that are listed.






The following XML files in the IBMConnections_root/version/history directory
describe fixes that are currently installed. These XML files
are related to installation items by the primary ID information, which is
indicated by italicized text like this.
 fix-id.efixDriver


Fix-driver defining information


fix-id.efixApplied


Fix installation details






Logs and component backups


IBM Connections products use two other
directories when performing update operations, for logging and backups. By
default, the two directories are relative to the product version directory,
as follows:
  
IBMConnections_root/version/log


Product updates log directory. IBM Connections products
store log files to document component, interim fix operations
and updates.


IBMConnections_root/version/backup


Product updates backup directory. IBM Connections products
back up components before applying interim fixes. If you uninstall
an interim fix, IBM Connections products
restore the backed-up component JAR file.






Examples of log file names


Interim fix
log file names are named as follows:
timeStamp_fixId_operation.log

where
these naming conventions are used:
timeStamp


YYYYMMDD_HHMMSS
For example: 20020924_211832 is 24-Sep-2002, 9:18:32
pm, GMT. All time stamps are in GMT.




fixId


Interim fix ID
For example: PQ77683_LC100_iFix is an interim
fix ID.

 


operation


The operation is either install or uninstall
For example: IBMConnections_root/version/log/20030915_144857_PQ77683_LC100_iFix_install.log








Here are some examples of log file names for common
IBM Connections Update Installer operations:
Interim fix install and uninstall


timeStamp_fixId_operation.log

example: IBMConnections_root/version/log/20030915_144857_PQ77683_LC100_iFix_install.log
and IBMConnections_root/version/log/20030915_162140_PQ77683_LC100_iFix_uninstall.log

the corresponding backup JAR file is not present.
